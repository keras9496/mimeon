# 미먼 (MiMeon) — 공기는 뇌에 바로 영향을 준다

> 배포 URL: **https://mimeon.onrender.com/**
> 저장소: https://github.com/keras9496/mimeon
> 문서 최종 수정: 2026-05-03

## 1. 개요

**미먼(微微, MiMeon)** 은 사용자가 평일에 머무는 1~3 곳의 주 생활공간 정보를 입력하면, 지난 60일(전일까지) 동안 그 공간·시간에 노출된 PM2.5·NO₂의 **위험도 등급**을 계산해 시각화하는 웹 서비스입니다.

PM2.5와 NO₂는 최근 메타분석(Khreis et al., *Lancet Planetary Health* 2025)에서 치매·뇌졸중·파킨슨병 위험을 유의하게 높이는 것으로 확인된 두 가지 핵심 인자입니다. 미먼은 **"공기는 뇌에 바로 영향을 준다"** 라는 메시지로, 일반 사용자가 자신의 생활 공간이 갖는 신경학적 미세먼지 위험을 직관적으로 이해할 수 있도록 합니다.

**핵심 사용자 흐름**
1. 홈에서 평일 주 생활공간 1~3개를 박스에 등록 (실내/실외, 거주 시간대 입력)
2. 위치는 카카오맵 키워드 검색으로 선택 → 좌표 자동 입력
3. "지난 2달 미먼 위험도 검사하기" 버튼 클릭
4. 종합 등급(낮음·보통·높음·매우 높음) + 위치별 카드 + 카카오맵 색박스 시각화 결과 확인
5. 좌우 화살표로 위치 1↔2↔3 간 네비게이션 (확대된 맵 + 노출 경고 카드)

## 2. 주요 기능

| 기능 | 설명 |
|---|---|
| 주 생활공간 박스 (1~3) | 각 박스는 "실내/실외" 라벨 + 클릭 시 검색·시간 모달 오픈 |
| 카카오맵 키워드 검색 | `kakao.maps.services.Places` — 회사명·건물명·도로명·지번 모두 검색 가능 |
| 거주 시간 입력 | 시작/종료 시각 (0~23시) — 자정 넘김(예: 20→08) 자동 처리 |
| 실내/실외 토글 | 실내 박스에는 침투계수(0.90)를 적용해 외기 → 실내 노출 보정 |
| 60일 평일 시간대 분석 | 어제까지를 끝으로 60일, 평일(월~금), 사용자 지정 시간대만 필터 |
| PM2.5/NO₂ 임계 초과율 | PM2.5 ≥ 35 ㎍/㎥, NO₂ ≥ 0.06 ppm 초과 시간 비율로 등급 산출 |
| 위치별 등급 + 종합 등급 | 횟수는 비공개. 등급(낮음·보통·높음·매우 높음)·색상으로만 표현 |
| 카카오맵 결과 시각화 | 위치마다 약 666m × 666m 색 박스 + 50% 투명도 오버레이 (파랑 → 빨강) |
| 위치 간 네비게이션 | 좌우 화살표 + 하단 1/2/3 인디케이터 + 우상단 "전체 보기" 버튼 |
| 진입 시 위치 1 줌인 | 위치 1로 panTo + 줌 레벨 3 + 미먼 노출 경고 카드 자동 표시 |

## 3. 기술 스택

### Backend
- **Python 3.12.7** + **FastAPI 0.115**
- `httpx` — 비동기 외부 API 호출
- `numpy` — ECEF 좌표 변환 및 측정소 최근접 탐색
- `pandas` — 측정소 마스터 데이터 관리

### Frontend
- **React 18** + **TypeScript** + **Vite 5**
- **카카오맵 JavaScript SDK** + `services` 라이브러리 (검색 + 지도 모두)
- 외부 지도 라이브러리(Leaflet) 사용하지 않음 — 단일 SDK로 통일

### 데이터 & 배포
- 데이터 1: **에어코리아 OpenAPI** (한국환경공단, 공공데이터포털) — 측정값
- 데이터 2: **카카오 로컬 API** (Places, Geocoder) — 주소 검색
- 호스팅: **Render** (단일 Web Service, Free 플랜)
- 저장소: GitHub (public)

## 6. 위험도 산출 모델

### 6.1 임계값 (한국 환경부 24h "나쁨" 기준)
| 오염물질 | 임계 | 근거 |
|---|---|---|
| PM2.5 | ≥ 35 ㎍/㎥ | 환경부 대기환경기준 24h 나쁨 |
| NO₂ | ≥ 0.06 ppm | 환경부 대기환경기준 24h 나쁨 |

각 위치에서 사용자가 지정한 평일 시간대의 1시간 단위 측정값에 대해, 실내인 경우 침투계수를 곱한 후 임계 초과 여부를 판정합니다.

### 6.2 실내 침투계수
| 오염물질 | F_inf | 의미 |
|---|---|---|
| PM2.5 | **0.90** | 외기 농도의 90%가 실내에 그대로 침투 (10% 감쇠) |
| NO₂ | **0.90** | 동일 |

학술 권장 default(K-IOP 기준 PM2.5 0.55, NO₂ 0.70)는 한국 평균이지만, 실제 사용자 환경(노후 아파트의 누기, 빈번한 자연환기, 가스레인지·조리 발생원 등)에서는 노출이 외기에 가깝습니다. 위험을 과소평가하지 않는 보수적 방향으로 0.90 으로 보정했습니다 (사용자 피드백 반영, 2026-05-03).

### 6.3 위험도 점수 = 가중 임계 초과율
```
PM2.5 초과율(%) = (PM2.5 임계 초과 시간) / (PM2.5 측정값 있는 시간) × 100
NO₂   초과율(%) = (NO₂   임계 초과 시간) / (NO₂   측정값 있는 시간) × 100

가중 초과율(%) = (1.0 × PM2.5 초과율 + 0.5 × NO₂ 초과율) / 1.5
```

PM2.5 가중치를 NO₂의 2배로 둔 근거:
- Khreis 2025 메타분석에서 PM2.5는 5 ㎍/㎥ 증가당 치매 HR 1.08
- NO₂는 10 ㎍/㎥ 증가당 HR 1.03
- 단위 환산하면 PM2.5의 효과크기가 NO₂ 대비 약 2배 강함

### 6.4 등급 체계
| 가중 초과율 | 등급 | 색상(50% 투명도 오버레이) |
|---|---|---|
| < 5%  | 낮음 | 파랑 `#2563eb` |
| < 15% | 보통 | 청 `#0ea5e9` |
| < 35% | 높음 | 주황 `#f97316` |
| ≥ 35% | 매우 높음 | 빨강 `#dc2626` |

오염물질별 위험 레벨도 같은 임계값으로 PM2.5 / NO₂ 각각 산출 (시각 색상은 두 레벨 중 더 높은 쪽 채택).

### 6.5 종합 위험도
```
종합 점수 = Σ(위치별 점수 × 위치별 매칭시간) / Σ(매칭시간)
```
시간 가중평균이며, 가장 위험한 위치(`worst_location_name`)도 별도 표시.

### 6.6 횟수 비공개 원칙
임계 초과 시간(횟수)은 백엔드 응답에 `_pm25_high_count`, `_no2_high_count` 로 디버깅용으로 포함되지만, **프론트엔드에서는 노출하지 않음**. 일반 사용자가 횟수의 절대값을 잘못 해석할 수 있어 위험도 등급/색상으로만 표현합니다.

## 7. 핵심 처리 알고리즘

### 7.1 GPS → 최근접 측정소 매칭
- 위·경도를 ECEF(지구중심좌표계) 3D 벡터로 변환
- 전체 672개 측정소와 유클리드 거리 계산 (numpy 벡터 연산, ~1ms)
- 최단거리 측정소 선택 후 Haversine 공식으로 km 환산

### 7.2 시간 윈도우
- 종료 시각: **어제 23:59:59** (전일까지)
- 시작 시각: 종료에서 60일 전 + 1초
- 평일(월~금)만 포함
- 사용자 지정 `start_hour ~ end_hour` 범위만 필터
  - 자정 넘김(예: 22→6)은 `hour >= start OR hour < end` 로 처리

### 7.3 측정소 캐싱 + 비동기 호출
- 1~3 위치 → 최근접 측정소 (중복 제거)
- 측정소별 단 1회 API 호출, `asyncio.gather` 로 병렬화
- 60일치 시간 단위 데이터: `numOfRows=1000` × 2페이지 (최대 2000행 ≈ 83일 커버)

### 7.4 시간 매칭
- 에어코리아 `dataTime` 의 `24:00` 특수 표기는 다음날 `00:00` 으로 정규화
- 각 1시간 측정 행을 윈도우/평일/시간대 필터에 통과시킨 뒤 임계 초과 판정

### 8.1 v2 (현재 메인)

#### `POST /api/exposure/risk-report`

요청
```json
{

      "lat": 37.4979, "lon": 127.0276,
      "is_indoor": true,
      "start_hour": 9, "end_hour": 18
    }
  ]
}
```

응답 (요약)
```jsonc
{
  "window": {
    "start": "2026-03-04T00:00:00",
    "end":   "2026-05-02T23:59:59",
    "lookback_days": 60
  },
  "locations": [{
    "name": "주 생활 공간 1",
    "address": "강남역",
    "lat": 37.4979, "lon": 127.0276,
    "is_indoor": true,
    "start_hour": 9, "end_hour": 18,

    "pm25_avg": 22.23, "no2_avg": 0.0194,
    "_pm25_high_count": 74, "_no2_high_count": 0,  // 프론트에서 미노출
    "pm25_risk_level": 3, "no2_risk_level": 1,
    "risk_score": 13.2,
    "risk_grade": "보통",
    "infiltration_applied": true
```

제한: 최대 3개 위치, `start_hour`/`end_hour` 0~23.

## 9. UI 흐름

### 9.1 홈 화면 (`/`)
- 헤더 라벨 "미먼" + 큰 문구 **"공기는 뇌에 바로 영향을 준다"**
- 안내 카드: "내가 생활하는 위치 기준 간이 검사" 설명
- 박스 3개 (주 생활 공간 1·2·3)
  - 왼쪽: 실내/실외 라벨 (입력 전 회색 "실내/실외", 입력 후 파랑 "실내" 또는 초록 "실외")
  - 본체: 클릭 시 모달 오픈, 입력 완료 시 주소·시간 표시
- 큰 CTA 버튼 "지난 2달 미먼 위험도 검사하기"

### 9.2 검색 모달 (`PlaceSearchModal`)
- 위치 검색 입력란 + 검색 버튼
- `kakao.maps.services.Places.keywordSearch` 결과 리스트 (스크롤 영역)
- 클릭으로 선택 → 선택 정보 표시
- 실내/실외 토글 (큰 버튼 2개)
- 거주 시간: 시작/종료 시각 드롭다운 (0~23시)
- 자정 넘김 안내 텍스트
- 저장/취소

### 9.3 결과 화면
좌측 패널 (420px 고정)
- 종합 위험도 배지 (등급 색상 50% 투명 배경)
- 가장 위험한 위치 표시
- 위치별 카드 (등급 배지 + PM2.5/NO₂ 레벨 + 측정소·거리)
- 색상 범례 + 안내 텍스트

우측: 카카오맵 (`KakaoResultMap`)
- 위치마다 색박스(`Polygon`, ±0.003° ≈ 666m 변, 50% 투명도)
- 중앙 마커
- **진입 시 위치 1로 panTo + 레벨 3 줌인**
- 좌상단 "미먼 노출 경고" 카드 (현재 위치의 등급·실내실외·시간대·측정소)
- **좌우 화살표** (가장자리 중앙) — 위치 인덱스 ±1, 경계에서 비활성화

## 11. 배포 구성

### Render 설정 (`render.yaml`)
- **Runtime**: Python (Node.js 빌드도 동일 환경에서 수행)
- **Build Command**:
  `cd frontend && npm ci && npm run build && cd ../backend && pip install -r requirements.txt`
- **Start Command**:
  `cd backend && uvicorn app.main:app --host 0.0.0.0 --port $PORT`
- **Health Check**: `/api/health`
- **환경변수**:
  - `PYTHON_VERSION=3.12.7`
  - `AIRKOREA_API_KEY` — 에어코리아 서비스 키 (시크릿, sync:false)
  - `VITE_KAKAO_MAP_KEY` — 카카오 JS키 (빌드 시 인라인, sync:false)
  - `CORS_ORIGINS=https://mimeon.onrender.com`

### 빌드 파이프라인
1. Vite 가 `frontend/src/` → `frontend/dist/` 로 번들링 (~0.5s, **162 kB** js, gzip 53 kB)
   - v1 대비 약 50% 감소 (Leaflet 제거)
2. pip 가 백엔드 의존성 설치
3. Uvicorn 이 `backend/app/main.py` 실행
4. FastAPI 가 `/api/*` 라우트 등록 후 `frontend/dist` 를 `/` 에 StaticFiles 로 마운트

### 카카오 개발자 콘솔 사전 설정
1. 애플리케이션 생성 → **JavaScript Key** 발급
2. **앱 설정 → 플랫폼 → Web** 에 도메인 등록:
   - `https://mimeon.onrender.com` (운영)
   - `http://localhost:5173`, `http://localhost:8000` (개발)
3. **제품 설정 → 카카오맵 → 활성화 ON** (필수, 미활성화 시 SDK 자체가 404 반환)

## 12. 프라이버시 및 보안

- **사용자 입력 비저장**: 위치·시간 입력은 요청 처리 중에만 메모리에 존재, DB·로그에 저장 안 함
- **AIRKOREA_API_KEY 격리**: 백엔드 환경변수, 프론트 번들에 미포함
- **VITE_KAKAO_MAP_KEY 노출**: 본질적으로 공개 키 (브라우저 번들 인라인). 보안은 카카오 측 도메인 화이트리스트 + 카카오맵 서비스 활성화 토글로 보장
- 카카오 검색 호출은 사용자 브라우저 → `dapi.kakao.com` 직접 (백엔드 경유 없음)
- 에어코리아 호출은 백엔드 → 공공데이터포털 (사용자 IP/식별 정보 미포함)

## 13. 제약사항

- **조회 기간**: 에어코리아 실시간 API는 최근 3개월 이내만 가능. v2 는 그 안에서 최근 60일 윈도우만 사용
- **측정소 결측**: 일부 시점·측정소는 데이터가 결측일 수 있음 → 해당 시간은 카운트에서 제외, `matched_hours` 에 미포함
- **API 트래픽**: 공공데이터포털 활용신청 계정의 일일 트래픽 한도 (개발계정 ~500/일)
- **카카오 무료 한도**: 로컬 검색 30만/일 (사용자 직접 호출이라 백엔드 부담 없음)
- **Cold start**: Render 무료 플랜 기준 15분 유휴 시 슬립 (첫 요청 10~30초 지연)
- **시간대 가정**: 평일(월~금) 고정, 주말은 별도 입력 미지원
- **단일 사용자 가정**: 박스 3개는 한 사람의 1·2·3순위 공간이며, 다중 사용자 비교 미지원

## 16. 로컬 개발

### 백엔드
```bash
cd backend
python -m venv .venv
. .venv/Scripts/activate          # Windows
# source .venv/bin/activate       # macOS/Linux
pip install -r requirements.txt
python scripts/fetch_stations.py  # 최초 1회 — 측정소 정본 JSON 생성
uvicorn app.main:app --reload --port 8000
```

### 프론트엔드
```bash
cd frontend
cp .env.example .env             # VITE_KAKAO_MAP_KEY 입력
npm install
npm run dev
```
- Dev: http://localhost:5173 (API 는 localhost:8000 으로 자동 연결)
- 빌드: `npm run build` → `dist/` 생성 → 백엔드가 자동 서빙

### 환경변수 파일
**`.env` (레포 루트, 백엔드용)**
```
AIRKOREA_API_KEY=<에어코리아 서비스 키>
CORS_ORIGINS=http://localhost:5173
```

**`frontend/.env` (프론트 빌드용)**
```
VITE_KAKAO_MAP_KEY=<카카오 JavaScript 키>
```
두 파일 모두 `.gitignore` 에 포함됨.

### 카카오 키 로컬 사용
로컬 개발용 도메인(`http://localhost:5173`)을 카카오 개발자 콘솔의 플랫폼에 등록해야 SDK 가 로드됩니다. 등록되지 않은 도메인에서는 SDK 가 `OPEN_MAP_AND_LOCAL service disabled` 또는 도메인 거부 응답을 반환합니다.
